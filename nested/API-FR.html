Aperçu

Prêt à plonger dans l'API Beewant ? Avec notre plateforme polyvalente, vous pouvez facilement extraire des informations à partir de vidéos, fichiers audio et documents, transformant vos données en informations exploitables. Basée sur les principes REST, notre API renvoie des réponses au format JSON, la rendant accessible dans divers langages de programmation et outils comme Postman ou tout client REST.

Décomposition

  • Prise en charge des méthodes : Nous proposons plusieurs méthodes pour interagir avec l'API :
    • GET : Récupérer des données
    • POST : Créer quelque chose de nouveau ou déclencher une action
    • PATCH : Mettre à jour des informations existantes
    • DELETE : Supprimer quelque chose
  • URL de base : Notre API est accessible à https://beewant.com/api
  • Paramètres : Spécifiez ce avec quoi vous voulez interagir (vidéos, audio ou documents) en utilisant :
    • Paramètres de chemin : Pointer vers un objet spécifique
    • Paramètres de requête : Options supplémentaires pour filtrer ou trier vos réponses
  • Authentification : N'oubliez pas d'inclure votre clé API dans les en-têtes pour authentifier vos requêtes

Réponses et gestion des erreurs

Notre API suit la norme RFC 9110, vous saurez donc toujours si votre requête a réussi ou non. Voici une répartition des codes d'état HTTP courants :

  • 200 : Requête réussie
  • 201 : Nouvelle ressource créée
  • 400 : Mauvaise requête. Vérifiez vos champs code et message pour plus de détails
  • 401 : Non autorisé. Clé API invalide
  • 404 : Ressource non trouvée
  • 429 : Limite de débit dépassée

Si vous rencontrez une erreur 4xx, la réponse inclura un code et un message convivial expliquant le problème, qui peut être affiché dans votre application.

Exemples de flux de travail

Cette section fournit un aperçu des flux de travail courants pour utiliser la plateforme Beewant Multimodal AI. Chaque flux de travail comprend une série d'étapes, et vous pouvez vous référer à la documentation détaillée pour plus d'informations sur chacune.

Lors du téléchargement de contenu (vidéos, audio ou documents) sur la plateforme, un traitement asynchrone est requis. Veuillez attendre que le traitement du contenu soit terminé avant de passer aux étapes suivantes.

Prérequis

  1. Si vous avez déjà un compte, allez sur la page Clé API dans vos Comptes et Paramètres Beewant et cliquez sur l'icône Copier à côté de votre clé
  2. Si vous n'avez pas de compte, inscrivez-vous pour un compte Beewant, puis suivez les étapes ci-dessus pour récupérer votre clé API

Types de contenu pris en charge

  • Vidéos : Format MP4
  • Audio : Format MP3
  • Documents : Fichiers texte tels que PDF, TXT, JSON et DOC

Pour commencer

  1. Sélectionnez le(s) type(s) de contenu que vous souhaitez analyser
  2. Téléchargez votre contenu en utilisant les points de terminaison API appropriés
  3. Attendez que le traitement soit terminé. Un bouton Discussion apparaîtra lorsque c'est fait
  4. Utilisez le Playground de Discussion pour extraire des informations de votre contenu traité

Index pour n'importe quel type

Méthode API

POST api/index

Paramètres de requête
Paramètre Type Description
mediaFile File Le fichier à indexer. Remplacez file_path par le chemin complet vers votre fichier
mediaType String Le type de média. Peut être document, video ou audio
advanced Boolean Définir sur true pour activer l'indexation avancée, qui effectue une analyse de contenu approfondie du fichier téléchargé (texte, images, graphiques, etc.), rendant les éléments consultables et interactifs. Pour une indexation texte uniquement plus rapide, définissez ce paramètre sur false
Exemple de requête
import requests, mimetypes

# Remplacez par votre token API
url = "https://beewant.com/api/index"

# En-têtes avec token
headers = {
    "Authorization": "Token <VOTRE_TOKEN>"
}

# Paramètres de charge utile
payload = {
    "mediaType": "document",
    "advanced": False,
}

# Fichier à télécharger
file_path = "/chemin/vers/votre/exemple.pdf"
files = [
    (
        "mediaFile",
        (
            "exemple.pdf",
            open(file_path, "rb"),
            mimetypes.guess_type(file_path)[0] or "application/octet-stream",
        ),
    )
]

response = requests.post(url, headers=headers, data=payload, files=files)
print(response.text)
Schéma de réponse
Paramètre Type Description
id Integer Identifiant unique du fichier téléchargé
file_link String Le lien URL pour accéder au fichier téléchargé
title String Nom du média
user Integer ID utilisateur de la personne qui a téléchargé le fichier

Créer un index | URL vidéo

Méthode API

POST api/index-video

Corps de la requête
Paramètre Type Description
videoLink String Le lien URL vers la vidéo YouTube. Remplacez selon les besoins
Exemple de requête
import requests
import json

url = "https://beewant.com/api/index-video"

# Remplacez le lien Youtube par n'importe quel lien souhaité
video_link = "https://www.youtube.com/watch?v=fPL4kUuanrg"

# Changez pour votre Token
headers = {
    "Authorization": f"Token <VOTRE_TOKEN>",
    "Content-Type": "application/json",
}

data = {
    "videoLink": video_link,
}

response = requests.post(url, headers=headers, json=data)
response.json()

Collections

Les collections dans Beewant permettent aux utilisateurs de regrouper plusieurs médias indexés, comme des documents, des audios et des vidéos, dans des ensembles cohérents pour un accès et une interaction faciles. Cette fonctionnalité simplifie l'organisation du contenu, permettant aux utilisateurs de gérer, mettre à jour ou récupérer des médias connexes ensemble, améliorant les flux de travail collaboratifs et rationalisés.

Créer une collection

Méthode API

POST api/collection/create

Paramètres de requête
Paramètre Type Description
name String Le nom de la collection
documents List Une liste des ID de documents à inclure dans la collection
audios List Une liste des ID audio à inclure dans la collection
videos List Une liste des ID vidéo à inclure dans la collection
Exemple de requête
import requests
import json

url = "https://beewant.com/api/collection/create"

payload = json.dumps({
    "name": "<NOM_COLLECTION>",
    "documents": [
        <DOCUMENT_ID_1>,
        <DOCUMENT_ID_2>
    ],
    "audios": [
        <AUDIO_ID_1>
    ],
    "videos": [
        <VIDEO_ID_1>
    ]
})

headers = {
    'Authorization': f'Token <VOTRE_TOKEN>',
    'Content-Type': 'application/json'
}

response = requests.post(url, headers=headers, data=payload)
print(response.text)

Discussion

Beewant offre une expérience de discussion interactive et polyvalente, vous permettant d'interagir avec divers types de médias de manière transparente. Que vous discutiez de texte brut, analysiez des images, interagissiez avec des documents indexés ou exploriez des collections de médias, notre plateforme de discussion est conçue pour s'adapter à vos besoins et fournir des informations précieuses dans tous les formats.

Transcription

L'Agent de Transcription dans Beewant convertit le contenu vidéo ou audio en texte précis et accessible. Cette fonctionnalité est idéale pour générer des transcriptions pour la documentation, l'accessibilité ou la réutilisation du contenu dans des formats texte, garantissant une utilisabilité transparente dans divers scénarios.

Méthode API

POST api/transcription

Paramètres de requête
Paramètre Type Description
mediaKey String Le hash unique représentant le fichier multimédia (par exemple, vidéo ou audio) pour récupérer la transcription
Exemple de requête
import requests
import json

url = "https://beewant.com/api/transcription"

payload = json.dumps({
    "mediaKey": "<MEDIA_HASH>"
})

headers = {
    'Authorization': f'Token <VOTRE_TOKEN>',
    'Content-Type': 'application/json'
}

response = requests.post(url, headers=headers, data=payload)
print(response.text)

Sources

L'Agent Sources dans Beewant extrait des segments ciblés et significatifs des collections, des chronologies et des morceaux de documents. Que vous commenciez de zéro ou continuiez une session, cette fonctionnalité simplifie l'analyse et la récupération de contenu.

Méthode API

POST api/media_sources

Paramètres de requête
Paramètre Type Description
prompt String La demande ou requête spécifique que vous souhaitez faire
mediaType String Le type de média interrogé. Peut être "collection", "video", "audio" ou "document"
mediaID Integer L'ID unique du fichier multimédia
chatModel String Le modèle de discussion à utiliser
currentChat Integer L'ID de la session de discussion en cours (pour continuer une discussion existante)

Agents

La fonctionnalité de Création d'Agent dans Beewant vous permet de définir des Agents personnalisés avec des capacités spécifiques. Cela permet une automatisation sur mesure pour le traitement, l'analyse et les flux de travail de transformation de contenu sur vos ressources médiatiques.

Créer un agent

Méthode API

POST api/agents

Paramètres de requête
Paramètre Type Description
name String Nom/identifiant unique pour votre agent personnalisé
description String Description optionnelle de l'objectif et des capacités de l'agent
instructions String Instructions de traitement pour l'agent à exécuter
icon File Fichier d'icône optionnel (par exemple, PNG/SVG)
Exemple de requête
import requests
import json

url = "https://beewant.com/api/agents"

payload = {
    "name": "nom_agent",
    "description": "Description de l'agent",
    "instructions": "Instructions de l'agent"
}

files = [
    ('icon', ('icon.png', open('votre/chemin/icon', 'rb'), 'image/png'))
]

headers = {
    'Authorization': f'Token <VOTRE_TOKEN>'
}

response = requests.post(url, headers=headers, data=payload, files=files)
print(response.text)
