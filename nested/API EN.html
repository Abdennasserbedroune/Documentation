Overview

Ready to dive into the Beewant API? With our versatile platform, you can easily extract information from videos, audio files, and documents, turning your data into actionable insights. Built on REST principles, our API returns responses in JSON format, making it accessible across various programming languages and tools like Postman or any REST client.

Breaking It Down

  • Method Support: We offer several methods to interact with the API:
    • GET: Retrieve data.
    • POST: Create something new or trigger an action.
    • PATCH: Update existing information.
    • DELETE: Remove something.
  • Base URL: Our API can be accessed at https://beewant.com.com/api.
  • Parameters: Specify what you want to interact with (videos, audio, or documents) using:
    • Path Parameters: Point to a specific object.
    • Query Parameters: Additional options for filtering or sorting your responses.
  • Authentication: Don't forget to include your API key in the headers to authenticate your requests.

Responses and Error Handling

Our API follows the RFC 9110 standard, so you'll always know if your request was successful or not. Here's a breakdown of common HTTP status codes:

  • 200: Request successful.
  • 201: New resource created.
  • 400: Bad request. Check your code and message fields for more details.
  • 401: Unauthorized. Invalid API key.
  • 404: Resource not found.
  • 429: Rate limit exceeded.

If you encounter a 4xx error, the response will include a code and a friendly message explaining the issue, which can be displayed in your app.

Workflow Examples

This section provides an overview of common workflows for using the Beewant Multimodal AI Platform. Each workflow includes a series of steps, and you can refer to the detailed documentation for more information on each.

When uploading content (videos, audio, or documents) to the platform, asynchronous processing is required. Please wait for content processing to complete before proceeding to the next steps.

Prerequisites

  1. If you already have an account, go to the API Key page in your Beewant Accounts & Settings and click the Copy icon next to your key.
  2. If you don't have an account, sign up for a Beewant account, then follow the steps above to retrieve your API key.

Supported Content Types

  • Videos: MP4 format.
  • Audio: MP3 format.
  • Documents: Text-based files such as PDF, TXT, JSON, and DOC.

Each content type undergoes specific processing to extract relevant information and insights. The platform's multimodal capabilities allow for integrated analysis across different media types, providing a comprehensive understanding of your data.

Getting Started

  1. Select the content type(s) you wish to analyze.
  2. Upload your content using the appropriate API endpoints.
  3. Wait for the processing to complete. A Chat button will appear when it's done.
  4. Use the Chat Playground to extract insights from your processed content.

Chat with tools

The Chat with tools feature in Beewant enables users to interact with the MCP (Model Context Protocol) system for advanced analysis and discussion. MCP is designed to handle a wide range of prompt types, including general questions, requests for up-to-date information, and even image generation, all within a unified chat interface.

General Prompts: When you ask a standard question that does not require external information, MCP will provide a direct answer without referencing external sources.

Web-Connected Answers: For prompts that require up-to-date or complex information (such as "What are the latest updates on artificial intelligence regulations?"), MCP will search relevant sources and return a list of citations. These citations include details such as URLs, snippets, and titles, giving transparency to the information provided. After gathering the necessary information, MCP will generate a comprehensive answer based on these sources.

Image Generation: If your prompt requests the generation of an image (for example, "Generate an image of a modern office workspace"), MCP will respond with a link to the generated image. This is followed by an answer or description related to the image.

External Tool Integration: MCP can also be connected to external tools, allowing it to perform actions or retrieve information beyond its built-in capabilities, further expanding the range of tasks it can handle.

The response format from MCP adapts to your prompt. You may receive a simple answer, a list of sources with a synthesized response, or an image URL with accompanying text. This flexibility allows you to use MCP for a variety of use cases, from research and summarization to creative image generation, all through a single API endpoint.
